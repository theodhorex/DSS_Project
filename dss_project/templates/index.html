📖 Cara Membaca Decision Tree (klik untuk buka)

Struktur Tree:

  • Kotak di atas (Root): Pertanyaan pertama yang paling penting
  • Kotak di tengah (Internal Nodes): Pertanyaan lanjutan berdasarkan jawaban sebelumnya
  • Kotak di bawah (Leaf Nodes): Hasil prediksi akhir

Informasi di setiap kotak:

  • Kondisi (contoh: "Exam_Pressure <= 5.00"): Pertanyaan yang diajukan
  • Warna: Menunjukkan kelas mana yang paling dominan (semakin gelap = semakin yakin)
  • Angka di bawah: Jumlah data training yang masuk ke node ini
  • Persentase: Distribusi kelas di node ini (contoh: [30%, 50%, 20%])

Cara mengikuti alur:

  • Mulai dari kotak paling atas (root node)
  • Cek apakah nilai Anda memenuhi kondisi (TRUE/FALSE)
  • Ikuti garis ke kiri (jika TRUE/<=) atau ke kanan (jika FALSE/>)
  • Ulangi sampai mencapai kotak paling bawah (leaf node) = hasil prediksi

Jalur Keputusan Anda:

Di bawah setiap tree, ada "Jalur Keputusan" yang menunjukkan step-by-step keputusan yang diambil model untuk data Anda. Ini adalah penjelasan tekstual dari alur yang Anda ikuti di tree visual.

Bagaimana hasil prediksi didapat

  • Input kamu diproses (imputasi dan encoding) agar sesuai format model.
  • Model membaca tree dari root, mengecek kondisi, lalu mengikuti cabang kiri/kanan.
  • Node terakhir (leaf) menentukan kelas hasil prediksi untuk tiap target.
  • Probabilitas dihasilkan dari distribusi kelas di leaf sehingga terlihat tingkat keyakinan.

Isi dan arti elemen di decision tree

  • Kondisi: aturan split, misalnya "Exam_Pressure <= 7.50".
  • Samples: jumlah data training yang masuk ke node tersebut.
  • Value: distribusi kelas pada node (untuk menghitung probabilitas).
  • Class: kelas dominan di node (hasil prediksi jika leaf).

          Penjelasan Decision Tree

          Apa itu Decision Tree?

          Decision Tree adalah model machine learning yang membuat keputusan dengan cara mirip seperti diagram alur. Model ini mengajukan pertanyaan beruntun tentang data Anda (contoh: "Apakah Exam_Pressure > 7.50?") dan berdasarkan jawaban, model akan terus menggali lebih dalam hingga mencapai kesimpulan akhir.

          Bagaimana cara kerjanya?

          • Root Node (Akar): Pertanyaan pertama yang paling penting untuk membedakan data
          • Internal Nodes (Cabang): Pertanyaan-pertanyaan lanjutan berdasarkan jawaban sebelumnya
          • Leaf Nodes (Daun): Kesimpulan akhir atau prediksi hasil
          • Split: Pembagian data berdasarkan kondisi tertentu (contoh: <= 50 atau > 50)

          Keuntungan Decision Tree:

          • Mudah dipahami dan dijelaskan (interpretable)
          • Dapat menunjukkan alasan keputusan step-by-step
          • Tidak perlu normalisasi data khusus
          • Cocok untuk data dengan banyak kategori

          Dalam sistem ini:

          • Ada 4 Decision Tree terpisah untuk 4 target berbeda (Stress Level, Anxiety Level, Final State, Intervention Response)
          • Setiap tree belajar dari 10,000 data training
          • Visualisasi tree ditampilkan di bagian "Visualisasi Pohon Keputusan"
          • Jalur keputusan Anda ditampilkan di "Faktor Utama" section

          Sumber Data: Dilatih menggunakan sklearn DecisionTreeClassifier dengan data training yang sudah diproses dan dinormalisasi.

          💡 Tips membaca Decision Tree:

          • Ikuti alur dari atas (root) ke bawah sesuai dengan kondisi Anda
          • Setiap node menunjukkan pertanyaan dan nilai input Anda
          • Warna node menunjukkan kelas mana yang paling dominan di node tersebut
          • Angka di node menunjukkan berapa banyak data training yang masuk ke node itu
          • Lihat "Jalur Keputusan" di bawah tree untuk penjelasan step-by-step

          Faktor Utama

          Penjelasan: Faktor utama adalah langkah-langkah keputusan pertama yang diambil model Decision Tree berdasarkan input Anda. Ini menunjukkan kondisi mana yang paling mempengaruhi hasil prediksi tingkat stres.

          Sumber Data: Diekstrak dari decision path tree untuk target "Stress Level" dan "Final State".

            Tahapan Preprocessing

            Penjelasan: Preprocessing adalah tahapan persiapan data sebelum masuk ke model machine learning. Ini termasuk:

            • Step 1 - Input Values: Nilai-nilai mentah yang Anda input dari form
            • Step 2 - Feature Preprocessing: Data setelah melalui normalisasi (scaling), imputation (mengisi missing values), dan encoding (mengubah kategori menjadi angka)

            Sumber Data: Diproses oleh sklearn ColumnTransformer dengan SimpleImputer dan OneHotEncoder.

            Probabilitas Prediksi

            Penjelasan: Probabilitas prediksi menunjukkan tingkat keyakinan model untuk setiap kemungkinan output. Semakin tinggi persentase, semakin yakin model dengan prediksi tersebut.

            Contoh: Jika "Stress Level" menunjukkan High: 53%, Medium: 35%, Low: 12%, berarti model 53% yakin Anda mengalami stres tinggi.

            Sumber Data: Dihitung dari predict_proba() method DecisionTreeClassifier untuk setiap target output.

            Rekomendasi Aksi

            Penjelasan: Rekomendasi aksi adalah saran tindakan praktis berdasarkan tingkat stres yang diprediksi. Setiap level stres memiliki rekomendasi berbeda:

            • Low: Pertahankan rutinitas yang sudah baik
            • Medium: Atur ulang prioritas dan tambah relaksasi
            • High: Kurangi aktivitas non-prioritas dan cari dukungan
            • Critical: Segera cari dukungan profesional

            Sumber Data: Berdasarkan mapping level stres ke ACTION_RECOMMENDATIONS yang telah ditentukan.
